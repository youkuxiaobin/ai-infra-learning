# 02 性能模型与逐元素优化

我们已经能写 GPU `vector add`，但性能是否还能提高？答案是：先要拆解总耗时、找瓶颈。

总耗时可拆为：

```text
T_total = T_H2D + T_kernel + T_D2H
```

当 `HtoD/DtoH` 远大于 `kernel` 时，`CPU -> GPU` 传输带宽可能是瓶颈。

这节课的核心方法论就一句话：

```text
先建模（Roofline / 算强度）
-> 判断瓶颈（memory-bound vs compute-bound）
-> 针对瓶颈做逐元素优化（对齐 / 合并 / 向量化 / half2）
-> 用 ncu 验证
```

## 1. 为什么“看起来跑了 GPU 但还是不快”

- 优化该从哪里下手：算术？线程数？方式？数据类型？
- 怎么用工具验证“我到底卡在带宽还是算力”？

## 2. 性能模型：Roofline

GPU kernel 性能上限由两条线约束：

- 算力上限：最多每秒做多少 FLOPs（compute peak）
- 带宽上限：最多每秒搬多少字节（memory bandwidth）

最终性能：

```text
Performance = min(算力上限能给到, 带宽上限能给到)
```

当处于 `memory-bound` 的时候，需要增加计算强度（AI）。

### 计算强度 AI

```text
AI = FLOPs / Bytes
```

也就是每搬 1B 数据，你能做多少计算。

对于逐元素 `float` 加法：

- 每个元素：读 `a(4B)` + 读 `b(4B)` + 写 `c(4B)` = `12B`
- 计算：`1 FLOP`（一次加法）

所以：

```text
AI = 1 / 12 = 0.083 FLOP / Byte
```

AI 很低，高度可能是 `memory-bound`。

## 3. Roofline 模型（看拐点，判断瓶颈）

对逐元素算子（`add / mul / relu / sigmoid ...`）来说，普遍特点是：

- 每个元素的计算很少（`1 ~ 数个 FLOP`）
- 但必须读写全局内存（至少 `8B ~ 12B`）

所以只要你访存效率没吃满带宽，就很难快。

一个常用判断流程是：

1. 给定 kernel
2. 计算 `AI = FLOPs / Bytes`
3. 判断 `AI` 是否小于 ridge point
4. 如果小于：优先做 `memory-bound` 优化（访存 / 带宽利用率）
5. 如果大于：优先做 `compute-bound` 优化（算力 / 并行度 / 指令效率）

### 对应优化方向

**Memory-bound：**

- 向量化 `load/store`
- 对齐与合并访问（coalescing）
- 使用 `half / half2` 降低 bytes

**Compute-bound：**

- 算子融合（operator fusion）
- 减少分支
- 利用 Tensor Core 等

## 4. 访存优化核心：对齐 + 合并

### warp 合并访存

- warp 有 32 个线程
- 理想情况：这 32 个线程访问连续地址
- GPU 才能把它们合成较少的 memory transaction

### `float3` 为什么常常是坑

- `float2 = 8B`，`float4 = 16B`：容易对齐、容易合并
- `float3 = 12B`：不友好，容易让访问跨事务边界，事务数上升，带宽利用下降

一个直观对比：

#### 糟糕：步长怪 / 未对齐 / `float3`

```text
T0 -> addr 0
T1 -> addr 12
T2 -> addr 24
...
T31 -> addr 372
```

结果：更多事务，带宽利用低。

#### 理想：连续且对齐

```text
T0 -> addr 0
T1 -> addr 4
T2 -> addr 8
...
T31 -> addr 124
```

结果：少量事务，带宽利用高。

## 5. 代码：从 baseline 到 vectorize 再到 half2

```cpp
#include <cuda_runtime.h>
#include <cuda_fp16.h>
#include <cstdio>
#include <vector>
#include <cassert>
#include <cmath>

#define CUDA_CHECK(call) do { \
    cudaError_t err = (call); \
    if (err != cudaSuccess) { \
        fprintf(stderr, "CUDA error %s:%d: %s\n", \
                __FILE__, __LINE__, cudaGetErrorString(err)); \
        std::exit(1); \
    } \
} while (0)

// -----------------------------
// 1) baseline: float 标量逐元素 add
// 每线程处理 1 个元素：最直观，但带宽可能吃不满
// -----------------------------
__global__ void add_f32(const float* a, const float* b, float* c, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) c[idx] = a[idx] + b[idx];
}

// -----------------------------
// 2) vectorized: float4
// 关键点：一次 load/store 16B，通常更容易吃满带宽（前提：对齐 + 连续）
// 注意：n4 = n/4，且 n 必须能被 4 整除（否则要单独处理尾巴）
// -----------------------------
__device__ __forceinline__ float4 add4(float4 x, float4 y) {
    return make_float4(x.x + y.x, x.y + y.y, x.z + y.z, x.w + y.w);
}

__global__ void add_f32x4(const float4* a, const float4* b, float4* c, int n4) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n4) c[idx] = add4(a[idx], b[idx]);
}

// -----------------------------
// 3) fp16: half
// 降低 bytes：每元素 2B，读+读+写=6B（理论上比 float 更省带宽）
// -----------------------------
__global__ void add_f16(const half* a, const half* b, half* c, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) c[idx] = __hadd(a[idx], b[idx]);
}

// -----------------------------
// 4) fp16 vector: half2
// half2 = 2 个 half 打包在 32-bit 寄存器里
// 优势：更少指令 + 更容易向量化 load/store（4B 一次）
// n2 = n/2，n 必须能被 2 整除
// -----------------------------
__global__ void add_f16x2(const half2* a, const half2* b, half2* c, int n2) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n2) c[idx] = __hadd2(a[idx], b[idx]);
}

// -----------------------------
// 计时工具：cudaEvent
// -----------------------------
float time_kernel(void (*launch)(int blocks, int threads), int iters = 200) {
    cudaEvent_t start, stop;
    CUDA_CHECK(cudaEventCreate(&start));
    CUDA_CHECK(cudaEventCreate(&stop));

    // warmup
    launch(1, 1);
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaEventRecord(start));
    for (int i = 0; i < iters; ++i) {
        launch(0, 0); // 让 launch 自己决定 blocks/threads
    }
    CUDA_CHECK(cudaEventRecord(stop));
    CUDA_CHECK(cudaEventSynchronize(stop));

    float ms = 0.f;
    CUDA_CHECK(cudaEventElapsedTime(&ms, start, stop));

    CUDA_CHECK(cudaEventDestroy(start));
    CUDA_CHECK(cudaEventDestroy(stop));
    return ms / iters;
}

// -----------------------------
// 主程序：跑四种实现并输出 GB/s
// -----------------------------
int main() {
    const int N = 1 << 24; // 数据规模：建议大一点，否则 malloc/launch 开销淹没 kernel
    const int threads = 256;

    // -----------------------------
    // float32 数据
    // -----------------------------
    std::vector<float> h_a(N, 1.0f), h_b(N, 2.0f), h_c(N, 0.0f);
    float *d_a=nullptr, *d_b=nullptr, *d_c=nullptr;
    CUDA_CHECK(cudaMalloc(&d_a, N * sizeof(float)));
    CUDA_CHECK(cudaMalloc(&d_b, N * sizeof(float)));
    CUDA_CHECK(cudaMalloc(&d_c, N * sizeof(float)));
    CUDA_CHECK(cudaMemcpy(d_a, h_a.data(), N * sizeof(float), cudaMemcpyHostToDevice));
    CUDA_CHECK(cudaMemcpy(d_b, h_b.data(), N * sizeof(float), cudaMemcpyHostToDevice));

    // baseline f32
    auto launch_f32 = [&](int, int){
        int blocks = (N + threads - 1) / threads;
        add_f32<<<blocks, threads>>>(d_a, d_b, d_c, N);
    };
    float ms_f32 = time_kernel((void(*)(int,int))(+[](int,int){})); // 占位避免复杂函数指针

    // 直接计时（更清晰）
    cudaEvent_t s,e;
    CUDA_CHECK(cudaEventCreate(&s));
    CUDA_CHECK(cudaEventCreate(&e));

    CUDA_CHECK(cudaEventRecord(s));
    for (int i = 0; i < 200; i++) { launch_f32(0,0); }
    CUDA_CHECK(cudaEventRecord(e));
    CUDA_CHECK(cudaEventSynchronize(e));
    float total_ms = 0;
    CUDA_CHECK(cudaEventElapsedTime(&total_ms, s, e));
    ms_f32 = total_ms / 200;
    CUDA_CHECK(cudaEventDestroy(s));
    CUDA_CHECK(cudaEventDestroy(e));

    // 拷回校验
    CUDA_CHECK(cudaMemcpy(h_c.data(), d_c, N * sizeof(float), cudaMemcpyDeviceToHost));
    for (int i = 0; i < 10; ++i) assert(std::fabs(h_c[i] - 3.0f) < 1e-6);

    // 估算带宽：每元素 12B（读4+读4+写4）
    double gbps_f32 = (double)N * 12.0 / (ms_f32 * 1e6);
    printf("[f32 ] time=%.4f ms, est=%.2f GB/s\n", ms_f32, gbps_f32);

    // -----------------------------
    // float4 vectorized（要求 N % 4 == 0）
    // -----------------------------
    assert(N % 4 == 0);
    int N4 = N / 4;
    const float4* d_a4 = reinterpret_cast<const float4*>(d_a);
    const float4* d_b4 = reinterpret_cast<const float4*>(d_b);
    float4* d_c4 = reinterpret_cast<float4*>(d_c);

    auto launch_f32x4 = [&](){
        int blocks = (N4 + threads - 1) / threads;
        add_f32x4<<<blocks, threads>>>(d_a4, d_b4, d_c4, N4);
    };

    CUDA_CHECK(cudaEventCreate(&s));
    CUDA_CHECK(cudaEventCreate(&e));
    CUDA_CHECK(cudaEventRecord(s));
    for (int i = 0; i < 200; i++) { launch_f32x4(); }
    CUDA_CHECK(cudaEventRecord(e));
    CUDA_CHECK(cudaEventSynchronize(e));
    CUDA_CHECK(cudaEventElapsedTime(&total_ms, s, e));
    float ms_f32x4 = total_ms / 200;
    CUDA_CHECK(cudaEventDestroy(s));
    CUDA_CHECK(cudaEventDestroy(e));

    double gbps_f32x4 = (double)N * 12.0 / (ms_f32x4 * 1e6);
    printf("[f32x4] time=%.4f ms, est=%.2f GB/s\n", ms_f32x4, gbps_f32x4);

    // -----------------------------
    // fp16 / fp16x2：为了公平，单独分配 half 数据
    // -----------------------------
    std::vector<half> h_ha(N), h_hb(N), h_hc(N);
    for (int i = 0; i < N; i++) {
        h_ha[i] = __float2half(1.0f);
        h_hb[i] = __float2half(2.0f);
    }

    half *d_ha=nullptr, *d_hb=nullptr, *d_hc=nullptr;
    CUDA_CHECK(cudaMalloc(&d_ha, N * sizeof(half)));
    CUDA_CHECK(cudaMalloc(&d_hb, N * sizeof(half)));
    CUDA_CHECK(cudaMalloc(&d_hc, N * sizeof(half)));
    CUDA_CHECK(cudaMemcpy(d_ha, h_ha.data(), N * sizeof(half), cudaMemcpyHostToDevice));
    CUDA_CHECK(cudaMemcpy(d_hb, h_hb.data(), N * sizeof(half), cudaMemcpyHostToDevice));

    auto launch_f16 = [&](){
        int blocks = (N + threads - 1) / threads;
        add_f16<<<blocks, threads>>>(d_ha, d_hb, d_hc, N);
    };

    CUDA_CHECK(cudaEventCreate(&s));
    CUDA_CHECK(cudaEventCreate(&e));
    CUDA_CHECK(cudaEventRecord(s));
    for (int i = 0; i < 200; i++) { launch_f16(); }
    CUDA_CHECK(cudaEventRecord(e));
    CUDA_CHECK(cudaEventSynchronize(e));
    CUDA_CHECK(cudaEventElapsedTime(&total_ms, s, e));
    float ms_f16 = total_ms / 200;
    CUDA_CHECK(cudaEventDestroy(s));
    CUDA_CHECK(cudaEventDestroy(e));

    // half 每元素 6B（读2+读2+写2）
    double gbps_f16 = (double)N * 6.0 / (ms_f16 * 1e6);
    printf("[f16 ] time=%.4f ms, est=%.2f GB/s\n", ms_f16, gbps_f16);

    // half2
    assert(N % 2 == 0);
    int N2 = N / 2;
    const half2* d_ha2 = reinterpret_cast<const half2*>(d_ha);
    const half2* d_hb2 = reinterpret_cast<const half2*>(d_hb);
    half2* d_hc2 = reinterpret_cast<half2*>(d_hc);

    auto launch_f16x2 = [&](){
        int blocks = (N2 + threads - 1) / threads;
        add_f16x2<<<blocks, threads>>>(d_ha2, d_hb2, d_hc2, N2);
    };

    CUDA_CHECK(cudaEventCreate(&s));
    CUDA_CHECK(cudaEventCreate(&e));
    CUDA_CHECK(cudaEventRecord(s));
    for (int i = 0; i < 200; i++) { launch_f16x2(); }
    CUDA_CHECK(cudaEventRecord(e));
    CUDA_CHECK(cudaEventSynchronize(e));
    CUDA_CHECK(cudaEventElapsedTime(&total_ms, s, e));
    float ms_f16x2 = total_ms / 200;
    CUDA_CHECK(cudaEventDestroy(s));
    CUDA_CHECK(cudaEventDestroy(e));

    // half2 仍然按 half 元素计 bytes：每 half 元素 6B
    double gbps_f16x2 = (double)N * 6.0 / (ms_f16x2 * 1e6);
    printf("[f16x2] time=%.4f ms, est=%.2f GB/s\n", ms_f16x2, gbps_f16x2);

    // cleanup
    CUDA_CHECK(cudaFree(d_a));
    CUDA_CHECK(cudaFree(d_b));
    CUDA_CHECK(cudaFree(d_c));

    CUDA_CHECK(cudaFree(d_ha));
    CUDA_CHECK(cudaFree(d_hb));
    CUDA_CHECK(cudaFree(d_hc));

    return 0;
}
```

## 8. 本节要点总结

- 不要只看“用了 GPU 没有”，而要先拆总耗时：`H2D / kernel / D2H`
- 用 Roofline 判断瓶颈，本质是比较 AI 和硬件 ridge point
- 逐元素算子通常 AI 很低，容易成为 `memory-bound`
- 访存优化核心是：**对齐 + 合并**
- `float4`、`half`、`half2` 的价值，本质上都是为了更好地利用带宽或减少 bytes
- 最后要用 profiling 工具验证，不要只凭感觉优化
